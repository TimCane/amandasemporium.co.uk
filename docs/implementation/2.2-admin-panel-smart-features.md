# 2.2 Admin Panel — Smart Features

**Phase:** 2 — Admin Panel

**Goal:** Enhanced admin panel features: AI content generation, location geocoding, photo auto-processing, and content validation dashboard.

## User Stories

- As a content owner, I want to generate a tagline and narrative for a bear with one click so I don't have to write everything from scratch.
- As a content owner, I want to provide a bit of context (e.g., "found at a car boot sale, was missing an ear") and get a personalised narrative.
- As a content owner, I want to search for a location by name and have the coordinates filled in automatically.
- As a content owner, I want to see which content is missing taglines, narratives, or other fields so I know what needs enriching.
- As a content owner, I want uploaded photos to be automatically resized and optimised without manual image editing.

## AI Content Generation

### How It Works

A "Generate" button on bear, product, and event forms runs `claude -p` (Claude Code CLI in headless mode) to generate content. Bears and products get `{ "tagline": "...", "narrative": "..." }`. Events get `{ "narrative": "..." }` only (visitor info is hand-typed).

### Flow

1. User fills in structured data (name, species, brand, location, dates, tags)
2. User optionally adds context in a "Generation context" text input (e.g., "found at a car boot sale, was wearing a tiny hat")
3. User clicks "Generate tagline & narrative"
4. Admin panel constructs a prompt from the structured data + context
5. Runs `claude -p "{prompt}"` as a child process
6. Parses the JSON response
7. Populates the tagline field and narrative textarea
8. User reviews, edits if needed, then saves

### Prompt Construction

```typescript
// Example for bears
const prompt = `Generate a JSON object with "tagline" and "narrative" fields for a rescued teddy bear.

Bear details:
- Name: ${name}
- Species: ${species}
- Brand: ${brand}
- Rescued from: ${rescueLocation} on ${rescueDate}
${rehomed ? `- Rehomed to: ${rehomeLocation} on ${rehomeDate}` : '- Still looking for a home'}
${event ? `- Shown at event: ${eventName}` : ''}
${tags.length ? `- Tags: ${tags.join(', ')}` : ''}
${context ? `- Additional context: ${context}` : ''}

Rules:
- "tagline": A short, warm one-liner (under 100 chars) for cards and previews
- "narrative": 2-3 paragraphs telling this bear's story in a warm, personal tone. Include where they were found, their personality, and where they ended up (if rehomed).
- Write in third person
- Be warm and whimsical but not saccharine
- Both fields should be tonally consistent

Respond with ONLY valid JSON, no markdown fences.`;
```

### Technical

- Uses Node.js `child_process.execSync` or `spawn` to run `claude -p`
- Relies on the user having Claude Code installed and authenticated (they do — they're running this project with it)
- No API keys needed — uses existing CLI auth
- Timeout handling (AI generation may take a few seconds)
- Error handling if `claude` CLI is not available or returns invalid JSON

## Location Geocoding

### How It Works

When adding or editing a location, a search field lets you type a place name. It queries the OpenStreetMap Nominatim API and auto-fills latitude/longitude, country, county, and postcode district.

### Flow

1. User types a place name (e.g., "Guildford, UK")
2. Debounced search (300ms) queries Nominatim
3. Dropdown shows matching results with full address
4. User selects one
5. Latitude, longitude, country, county, and postcode fields auto-populate
6. User can manually adjust if needed

### Technical

- API: `https://nominatim.openstreetmap.org/search?q={query}&format=json&countrycodes=gb`
- No API key needed
- Rate limit: 1 request/second (debounce handles this)
- `countrycodes=gb` defaults to UK results (most locations are UK-based)
- Fallback: manual lat/lng entry always available

### Also Used In

- Bear create/edit form — when selecting rescue/rehome location, if the location doesn't exist, offer to create a new one inline with geocoding

## Photo Auto-Processing

### How It Works

When a photo is uploaded via the admin panel, it is automatically processed before saving to the content directory.

### Processing Steps

1. **Resize** — max 1200px on longest edge (preserves aspect ratio)
2. **Format** — convert to JPEG if not already (Astro's pipeline handles WebP/AVIF at build time)
3. **Rename** — rename to match the content slug (e.g., `aaron.jpg` for bear Aaron)
4. **Strip EXIF** — remove metadata (GPS location, camera info) for privacy
5. **Optimise** — basic JPEG quality optimisation (85% quality)

### Technical

- Uses `sharp` (already an Astro dependency) for all image processing
- Processing happens server-side in the admin route handler
- Original uploaded file is not kept — only the processed version

## Content Validation Dashboard

### How It Works

A dashboard page at `/admin/dashboard` (or section on the main `/admin` page) showing content completeness across all collections.

### Displays

| Check | What It Shows |
| ----- | ------------- |
| Bears missing tagline | Count + list of bear names |
| Bears with empty body (no narrative) | Count + list |
| Bears with empty tags | Count + list |
| Bears with single photo only | Count (not necessarily a problem, just info) |
| Products missing tagline | Count + list |
| Products with empty body | Count + list |
| Products missing "How It's Made" | Count + list (check for `## How It's Made` heading in body) |
| Products missing materials | Count + list |
| Events with empty body | Count + list |
| Past events missing recap | Count + list (past events without `## Recap` in body) |
| Gallery items missing category | Count + list |

### Visual

- Traffic light indicators: green (>80% complete), amber (50-80%), red (<50%)
- Click any count to see the list of items, click an item to jump to its edit form
- Overall content completeness percentage
