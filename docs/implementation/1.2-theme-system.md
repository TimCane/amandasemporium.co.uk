# 1.2 Theme System Foundation

**Phase:** 1 — Project Foundation

**Goal:** CSS Zen Garden architecture with 4 theme skeletons, theme registry, and switching mechanism.

## User Stories

- As a visitor, I want the site to load with the Boutique theme by default so it looks polished from the start.
- As a visitor, I want to switch between 4 themes that each feel like a completely different website.
- As a visitor, I want my theme choice remembered between visits.
- As a visitor, I want theme switching to be instant (no page reload, no flash).
- As a developer, I want a CSS Zen Garden architecture where components emit structural HTML and themes handle all visual design.

## Tasks

- [ ] Create `_base.scss` — base theme with sensible defaults for all HTML hooks (hidden decoration divs, default card/button/nav styles)
- [ ] Create theme SCSS file structure (`boutique.scss`, `whimsical.scss`, `modern.scss`, `fun.scss`) — each imports/extends base, overrides tokens and component styles
- [ ] Create `_shared.scss` with mixins and theme helpers
- [ ] Install all 8 fonts via `@fontsource` (4 heading + 4 body fonts)
- [ ] Create `src/libs/themes.ts` — theme registry with metadata (id, name, fonts, cssFile). Theme switcher, CSS loader, and font loader all read from this single registry.
- [ ] Implement inline `<script>` for flash prevention (reads localStorage before paint)
- [ ] Implement CSS loading strategy: active theme in `<head>`, others prefetched
- [ ] Implement theme stylesheet swap logic for switching
- [ ] Maintain `docs/theme-creation-guide.md` — documents all HTML hooks for future theme authors

## Notes

Theme SCSS files start as skeletons in Phase 1. They are fleshed out **in parallel with every component and page** in subsequent phases. Every time a component is built, its styling is implemented across all 4 themes simultaneously.

**Extensibility:** The base theme + registry pattern means adding a future theme requires only: 1) a new SCSS file, 2) installing fonts, 3) one entry in `themes.ts`. No component/layout/page changes.
