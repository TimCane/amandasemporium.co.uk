# 5.3b Event Profile Pages (Static)

**Phase:** 5 — Products & Events

**Goal:** Individual event pages with venue map, recap for past events, and bears-at-event section.

## User Stories

- As a visitor, I want to see full details about an upcoming event — what to expect, where it is, how to get there.
- As a visitor, I want to see a recap of how a past event went, with photos.
- As a visitor, I want to see which bears were at a particular event.

## Tasks

- [ ] Build `/events/[slug].astro` with `getStaticPaths` and HTML hooks
- [ ] Create event profile structure (header, narrative, venue map, recap, photos, decoration divs)
- [ ] For upcoming events: render markdown body as narrative, venue map (single Leaflet marker), website link, "Add to calendar" button
- [ ] For past events: render `## Recap` section from body, `recapPhotos` from frontmatter, "Bears at this event" section (bears referencing this event)
- [ ] Compute upcoming/past from date vs current date
- [ ] Integrate single-marker Leaflet map for venue
- [ ] Add SEO: OG tags, JSON-LD Event schema
- [ ] **Style event profile in all 4 themes**
- [ ] Update event cards on listing page to link to `/events/[slug]`
